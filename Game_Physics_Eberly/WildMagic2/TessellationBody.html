A new version of Delaunay triangulation in 2D. Hopefully this one is more robust than the other one. Comparison of test results with a version that uses exact arithmetic appears to indicate the new version is robust. The older version is a modification of Dave Watson's code (nnsort.c) and has a few #define quantities related to choice of supertriangle, use of jittering to break collinearities and cocircularities, and allocation of space to hold the triangle information. The supertriangle method is not guaranteed to always produce the convex hull. Jittering is not necessary if the algorithm is implemented carefully. Allocation should be dynamic as needed and should not require the user to recompile and rerun the program if the specified memory is insufficient. The new version address all these issues and should always produce a triangulation whose triangles form the convex hull of the input data. Run-time on an AMD 1.4Ghz processor with 100,000 points was 16 seconds:
